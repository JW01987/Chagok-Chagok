## 1. 서비스 컨셉

### 한 줄 요약

> 매달 자동으로 적립하고, 포트폴리오는 앱이 추천해주는 — 사회초년생을 위한 첫 투자 습관 앱

### 왜 차곡차곡인가

사회초년생 10명 중 7명은 투자를 해야 한다고 알고 있다. 하지만 실제로 시작한 사람은 절반도 안 된다.

| 장벽      | 문제                         | 차곡차곡의 해결                   |
| --------- | ---------------------------- | --------------------------------- |
| 모른다    | 뭘 사야 할지 모르겠다        | 포트폴리오 **추천**으로 고민 제거 |
| 귀찮다    | 매달 직접 챙기기 번거롭다    | **자동 적립**으로 수고 제거       |
| 안 보인다 | 소액은 변화가 없어 보여 포기 | **성장 시각화**로 동기 부여       |

### 핵심 철학

| 원칙   | 설명                                            |
| ------ | ----------------------------------------------- |
| 단순함 | 처음 쓰는 사람이 5분 안에 첫 적립 시작 가능     |
| 자동화 | 한 번 설정하면 이후엔 신경 쓰지 않아도 됨       |
| 투명함 | 내 돈이 어디에 어떻게 굴려지는지 쉬운 말로 표시 |
| 지속성 | 작은 성장도 크게 보여줘서 계속하게 만듦         |

## 2. 타겟 유저

### 주요 타겟

- **연령**: 22~32세
- **상황**: 취업 후 1~3년차, 첫 월급을 받기 시작한 직장인
- **특징**: 재테크 관심 있으나 투자 경험 없음, 소액(월 10~30만원) 투자 희망

### 페르소나

**페르소나 A — 김지은 (26세, 직장인 2년차)**

- 월급 240만원, 쓰고 나면 남는 게 없음
- ETF 얘기는 들었는데 뭔지 모름
- 목표: 3년 안에 결혼 자금 3,000만원

**페르소나 B — 이민준 (29세, 직장인 4년차)**

- 주식 손실 경험으로 트라우마, 리스크 낮은 자동화 원함
- 목표: 10년 후 1억 만들기

### 3-1. 온보딩

처음 앱을 켰을 때 회원가입 없이 3가지만 입력하면 즉시 시작.

| 입력 항목         | 설명                    | 예시      |
| ----------------- | ----------------------- | --------- |
| 월 투자 가능 금액 | 슬라이더 (5~100만원)    | 20만원    |
| 투자 성향         | 안전 / 균형 / 성장      | 균형      |
| 투자 목표         | 목돈 / 노후 / 자유 저축 | 목돈 마련 |

- 목표 금액 + 기간 입력 시 매달 적립 금액 자동 역산
- 성향 테스트 (3~5문항) 또는 직접 선택

---

### 3-2. 포트폴리오 추천 엔진

| 유형                 | 구성                                                       | 대상      | 예상 연수익률 |
| -------------------- | ---------------------------------------------------------- | --------- | ------------- |
| 올웨더 (All Weather) | 주식 30% / 장기채 40% / 중기채 15% / 금 7.5% / 원자재 7.5% | 안전      | 연 7~9%       |
| 주식-채권 혼합       | 주식 60% / 채권 40%                                        | 균형      | 연 8~10%      |
| 글로벌 성장형        | 국내 ETF 30% / 미국 ETF 50% / 신흥국 ETF 20%               | 성장      | 연 10~12%     |
| 한국 집중형          | 코스피200 ETF 70% / 국내 채권 30%                          | 국내 선호 | 연 6~8%       |

각 포트폴리오 상세 화면:

- 구성 비율 도넛 차트
- 10년 백테스트 수익률 그래프 (면책 고지 포함)
- 구성 ETF 목록 및 리스크 지표 (MDD, 변동성)
- 포트폴리오 비교 기능 (2개 나란히)

---

### 3-3. 자동 적립

| 설정      | 옵션                 | 기본값        |
| --------- | -------------------- | ------------- |
| 적립 날짜 | 매월 1~28일          | 월급날 다음날 |
| 적립 금액 | 고정 / 월급 연동 %   | 20만원 고정   |
| 적립 주기 | 매월 / 격월 / 매주   | 매월          |
| 적립 방식 | 일괄 / DCA 분할 매수 | 분할 매수     |

- **DCA**: 월 20만원 → 주 5만원 × 4회. 고점 매수 리스크 감소
- 적립일 3일 전 예수금 잔고 확인 및 부족 알림
- 이번 달만 건너뛰기 / 3개월 중지 선택 가능

---

### 3-4. 수익 현황 대시보드 (메인 홈)

```
총 투자 원금        현재 평가금액       총 수익
₩2,400,000   →   ₩2,631,000    +₩231,000 (+9.6%)
```

- 월별 수익률 바 차트 (최근 12개월)
- 적립 히스토리 테이블
- 목표 달성률 프로그레스 바
- 현재 포트폴리오 비율 vs 목표 비율 비교 도넛 차트

---

### 3-5. 미래 자산 시뮬레이션

입력: 월 적립금 / 예상 연수익률 / 투자 기간 (슬라이더)
출력: 예상 자산, 원금 vs 수익 복리 시각화, 서울 전세 비교 카드

추가 옵션:

- 월급 인상 시나리오 (연봉 인상률 슬라이더)
- 목표 역산: "3년 후 3,000만원이면 매달 얼마?"
- 인플레이션 반영 실질 수익률 (고급 옵션)

---

### 3-6. 알림 센터

| 알림 유형   | 내용                        | 기본값 |
| ----------- | --------------------------- | ------ |
| 적립 예정   | "3일 후 월 적립 실행돼요"   | ON     |
| 적립 완료   | "이번 달 적립 완료됐어요 ✓" | ON     |
| 예수금 부족 | "적립일 전 잔고 부족해요"   | ON     |
| 수익률 요약 | 주간 수익률 리포트          | 주 1회 |
| 리밸런싱    | 비율 ±5% 이상 이탈 시       | ON     |
| 목표 달성   | 25% / 50% / 75% / 100%      | ON     |
| 시장 이슈   | 급락·급등 10% 이상          | OFF    |

알림 채널: 푸시 / 카카오 알림 / 이메일 선택

---

### 3-7. 월간 리포트

- 매월 말 자동 발송
- 월간 수익률, 누적 현황, 다음 달 예정 금액
- 동일 포트폴리오 유저 익명 평균 비교
- 카카오톡 / 인스타 스토리 공유 카드 자동 생성

## 4. 게스트 모드 정책

### 설계 원칙

> 처음 앱을 켰을 때 가입 없이 충분히 가치를 경험하게 한 뒤, 자연스럽게 회원가입으로 유도한다.

### 게스트 허용 범위

| 기능                    | 게스트 가능 여부 | 비고                 |
| ----------------------- | ---------------- | -------------------- |
| 온보딩 (성향·목표 입력) | ✅               | 로컬 임시 저장       |
| 포트폴리오 추천 조회    | ✅               | 전체 4종             |
| 시뮬레이션 계산기       | ✅               | 전체 기능            |
| 대시보드 (샘플 데이터)  | ✅               | 실제 수익 아님 명시  |
| 적립 설정 저장          | ❌               | 회원가입 유도 트리거 |
| 계좌 연동               | ❌               | 회원 전용            |
| 푸시 알림 수신          | ❌               | 회원 전용            |
| 월간 리포트             | ❌               | 회원 전용            |

### 유도 전환 시나리오

1. 게스트가 적립 설정 저장 시도 → "지금 설정을 저장하고 자동 적립을 시작하려면 간단히 가입해요" 팝업
2. 시뮬레이션 결과 확인 후 → "이 계획대로 실제로 시작해볼까요?" CTA
3. 앱 3회 이상 재방문 시 → 가입 유도 배너 (설정에서 끄기 가능)

### 게스트 데이터 처리

- 게스트 입력 데이터는 기기 로컬에만 저장 (AsyncStorage 암호화)
- 회원가입 시 로컬 데이터 → 서버 마이그레이션 처리
- 앱 삭제 시 게스트 데이터 자동 소멸

## 6. 보안 정책

> 상용 금융 서비스 수준의 보안을 초기부터 설계에 반영한다. GitHub 공개 저장소를 사용하므로 시크릿 유출 방지가 최우선이다.

### 6-1. GitHub 소스코드 보안

| 항목          | 정책                                                        |
| ------------- | ----------------------------------------------------------- |
| 시크릿 관리   | **절대 하드코딩 금지**. 모든 키·비밀번호는 환경 변수로 분리 |
| `.env` 파일   | `.gitignore`에 반드시 포함. `.env.example`만 커밋           |
| AWS 자격증명  | AWS IAM Role 사용 (Access Key 코드 내 사용 금지)            |
| 시크릿 스캐닝 | GitHub Secret Scanning + **GitLeaks** CI 단계 적용          |
| 브랜치 보호   | `main` / `develop` 브랜치 직접 푸시 금지, PR + 리뷰 필수    |
| 의존성 취약점 | GitHub Dependabot 자동 PR 활성화                            |

### 6-2. 인증 / 인가

| 항목             | 정책                                              |
| ---------------- | ------------------------------------------------- |
| 인증 방식        | JWT (Access Token 15분 + Refresh Token 30일)      |
| 토큰 저장 (앱)   | expo-secure-store (Keychain/Keystore 기반 암호화) |
| 소셜 로그인      | OAuth2 (카카오, 애플 필수)                        |
| 비밀번호         | BCrypt 해싱 (cost factor 12 이상)                 |
| 로그인 실패 제한 | 5회 실패 시 계정 잠금 15분                        |
| 관리자 API       | 별도 인증 레이어 + IP 화이트리스트                |

### 6-3. 데이터 보안

| 항목          | 정책                                             |
| ------------- | ------------------------------------------------ |
| 통신          | HTTPS 전용 (TLS 1.2 이상). HTTP 요청 차단        |
| DB 암호화     | 민감 컬럼 (계좌번호, 주민번호 등) AES-256 암호화 |
| 로그          | 개인정보 / 금융 데이터 로그 마스킹 처리          |
| CORS          | 허용 오리진 화이트리스트로 명시                  |
| SQL Injection | JPA Prepared Statement 사용, 동적 쿼리 최소화    |
| Rate Limiting | API별 요청 제한 (로그인: 분당 10회 등)           |

### 6-4. 앱 보안

| 항목           | 정책                                       |
| -------------- | ------------------------------------------ |
| 루팅/탈옥 감지 | react-native-device-info 기반 감지         |
| 스크린샷 제한  | 민감 화면 (계좌 정보 등) 스크린샷 비활성화 |
| 앱 난독화      | ProGuard (Android) / Bitcode (iOS)         |
| 인증서 피닝    | v1.0 이후 적용 검토                        |

## 7. CI/CD 파이프라인

```
[개발자 Push / PR]
       ↓
[GitHub Actions — CI]
  • 코드 빌드
  • 단위 테스트 (JUnit / Jest)
  • 시크릿 스캔 (GitLeaks)
  • 정적 분석 (SonarQube / Checkstyle)
  • 의존성 취약점 스캔
       ↓
[PR Merge → develop 브랜치]
       ↓
[GitHub Actions — CD (개발 서버)]
  • Docker 이미지 빌드
  • AWS ECR Push
  • 개발 서버 자동 배포 (ECS / EC2)
       ↓
[QA / 테스트 완료 후 main 태그]
       ↓
[GitHub Actions — CD (운영 서버)]
  • 운영 서버 배포 (수동 승인 트리거)
  • Slack / 이메일 배포 완료 알림
```

### 브랜치 전략 (GitHub Flow 기반)

| 브랜치      | 용도                          | 배포 대상                   |
| ----------- | ----------------------------- | --------------------------- |
| `main`      | 운영 배포용. 태그 기반 릴리즈 | 운영 서버                   |
| `develop`   | 개발 통합. PR 머지 기점       | 개발 서버                   |
| `feature/*` | 기능 단위 개발                | -                           |
| `hotfix/*`  | 운영 긴급 패치                | main 직접 머지 후 운영 배포 |
| `release/*` | 릴리즈 준비                   | 스테이징 서버 (v1.0 이후)   |

### GitHub Actions 워크플로 구성

| 워크플로 파일          | 트리거             | 역할                     |
| ---------------------- | ------------------ | ------------------------ |
| `ci.yml`               | PR 생성 / 업데이트 | 빌드, 테스트, 보안 스캔  |
| `cd-dev.yml`           | develop 머지       | 개발 서버 자동 배포      |
| `cd-prod.yml`          | main 태그 생성     | 운영 서버 수동 승인 배포 |
| `dependency-check.yml` | 매일 자정          | Dependabot 취약점 리포트 |

### 테스트 커버리지 기준

| 단계 | 최소 커버리지               |
| ---- | --------------------------- |
| MVP  | 핵심 비즈니스 로직 50% 이상 |
| Beta | 70% 이상                    |
| v1.0 | 80% 이상 (금융 로직 90%+)   |

## 10. 운영 환경 구성

### 환경 분리

| 환경        | 용도                     | 서버                 |
| ----------- | ------------------------ | -------------------- |
| local       | 개발자 로컬 개발         | Docker Compose       |
| development | 통합 개발 / 기능 검증    | AWS EC2 (t3.small)   |
| staging     | 릴리즈 전 QA (v1.0 이후) | AWS EC2 (t3.medium)  |
| production  | 실 서비스                | AWS EC2 (t3.medium+) |

### 환경별 설정 관리

- 환경변수: `.env.{환경명}` 파일 로컬 유지, CI/CD에서 GitHub Secrets → 서버 주입
- DB: 환경별 독립 RDS 인스턴스 (개발 / 운영 완전 분리)
- 로그 레벨: 개발(DEBUG) / 운영(WARN 이상)
